## Executive Summary

**Strategic Shift:** From broad implementation (~105h) to focused systems work (~55h).

### Key Changes from v1.0 (Based on Review):

| Area | v1.0 | v2.0 | Rationale |
|------|------|------|-----------|
| **Auth** | Custom JWT + BFF | Clerk SDK for Go | Don't reinvent auth; use Clerk SDK |
| **Monitoring** | Tempo + Loki + Jaeger | Jaeger (keep) + Prometheus/Grafana | Best-in-class, not single-vendor |
| **Admin Console** | Full CRUD UI (25h) | **REMOVED** | CRUD doesn't prove seniority; use Prisma Studio |
| **Order** | Auth â†’ Monitoring | **LiteDebug â†’ Monitoring â†’ Integrations â†’ Auth** | Visual demo first, plumbing last |

## Approved Implementation Order

### Phase A: Observability & Debugging (The "Showcase")
1. **LiteDebug Console** - Event Trace Viewer reading from Redpanda/Kafka
2. **Grafana Dashboards** - Prometheus metrics + Jaeger for tracing

### Phase B: Enterprise Architecture
1. **Integration Framework** - Go `Integration` interface and Adapter pattern
2. **Mockoon** - For testing integrations locally

### Phase C: Polish
1. **Auth** - Clerk SDK integration (Go backend + Next.js frontend)
